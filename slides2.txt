TLS/SSL e HTTPS
❖ O Java Secure Socket Extension (JSSE) realiza uma camada por cima
do TCP que adiciona segurança à troca de dados entre duas entidades
❖A segurança é conseguida através de uma concretização do protocolo
TLS/SSL
❖ O JSSE oferece uma interface ao protocolo TLS sobre sockets TCP (não
existe suporte para outros protocolos como UDP)
❖Principais vantagens do TLS/SSL:
➢ suportado por um número elevado de servidores e browsers
➢ interface simples para cifra com criptografia híbrida (usa criptografia
assimétrica para distribuir uma chave secreta (simétrica) para ser
usada para cifrar os dados da comunicação entre duas entidades)
➢ bem adaptado a Web, uma vez que normalmente apenas os
servidores se autenticam através de um certificado, usando os clientes
outras formas de autenticação (ex., passwords)
“Java Security”, cap. 14

Truststores e Keystores (1)
❖ O JSSE introduz o conceito das truststores que são basicamente uma
keystore que guarda certos certificados
❖Partilham
➢ o mesmo formato
➢ são representadas programaticamente por um objecto KeyStore
➢ são geridas pelas keytool
❖ O servidor utiliza durante a autenticação a chave privada e o respetivo
certificado armazenados na keystore
❖O cliente para verificar o certificado do servidor (que recebe deste)
necessita do certificado da CA que emitiu e assinou o certificado do
servidor. O certificado da CA contém a chave pública deste e que permitirá
validar a assinatura do certificado do servidor. O CA é a raiz de todo o
processo de segurança. Este certificado é guardado na truststore.
❖ Se o servidor necessitar de verificar o certificado do cliente, então este
também necessita de uma truststore.

Truststores e Keystores (2)
❖ Embora a keystore e a truststore possam corresponder ao mesmo ficheiro,
normalmente é mais fácil a gestão se a keystore tiver apenas um alias
com o certificado/chave privada do servidor
❖ Por omissão é assumido que a truststore está em primeiro lugar em
$JREHOME/lib/security/jssecacerts, e em segundo lugar
$JREHOME/lib/security/cacerts
❖ As seguintes propriedades podem ser usadas
➢ javax.net.ssl.trustStore: sítio da truststore para o SSL/TLS
➢ javax.net.ssl.keyStore: sítio da keystore para o SSL/TLS
➢ javax.net.ssl.keyStorePassword: password da keystore (a password
que protege a chave privada deve ser igual)
❖Normalmente é aconselhado o uso do algoritmo RSA para assinatura
porque é o mais suportado pelos browsers

Classes Factory para a criação de sockets - Server
❖ Estas classes factory dão mais flexibilidade à criação de diferentes tipos de
sockets usados no sistema
➢ Ex.: subclasses das factories podem criar sockets com características
pré configuradas ou mesmo retornar sockets específicos
❖Pode ser usada tanto para a criação de sockets normais e sockets que
usam o SSL
❖As classes javax.net.SocketFactory e javax.net.ServerSocketFactory
produzem respectivamente sockets normais para o lado do cliente e
servidor
❖ A API do ServerSocketFactory é
➢ public static ServerSocketFactory getDefault( )
• devolve a server socket factory que produz sockets normais
➢ public ServerSocket createServerSocket(int port)
➢ public ServerSocket createServerSocket(int port, int backlog,
InetAddress addr)
• cria uma socket servidor que escuta num dado porto (com um
específico backlog e interface de rede)

public class SSLSimpleServer extends Thread {
private Socket sock;
public SSLSimpleServer(Socket s) { // construtor
sock = s;
}
public static void main(String[] args) throws Exception {
ServerSocketFactory ssf = SSLServerSocketFactory.getDefault( );
SSLServerSocket ss = (SSLServerSocket) ssf.createServerSocket(9096);
while (true) {
new SSLSimpleServer(ss.accept()).start( ); // uma thread por ligação
}
}

Sessões SSL (1)
❖ Quando se realiza uma ligação de uma socket SSL, esta fica associada a
uma sessão SSL que mantém um conjunto de informações administrativas
❖ Caso existam diversas ligações entre o mesmo cliente e servidor, então é
possível que estas partilhem a mesma sessão
❖ A classe javax.net.ssl.SSLSession representa esta informação, e objectos
desta classe podem ser obtidos através do seguinte método das
SSLSocket
➢ public SSLSession getSession( )
❖ A interface do SSLSession oferece os métodos
➢ public byte[] getId( )
• retorna o identificador associado a esta sessão
➢ public SSLSessionContext getSessionContext( )
• devolve o contexto da sessão
➢ public long getCreationTime( )
• devolve o instante em que a sessão foi criada

• instante em que a sessão foi acedida pela última vez (i.e., quando
